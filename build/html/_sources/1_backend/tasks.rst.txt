任务管理
===============

**CCLAnnotator** 平台的任务管理除却文件操作，还需要考虑任务指派和任务状态更新。这也是控制器 ``TasksController`` \
的目的。与用户管理一样，首先需要为之设计一个任务模型，路径为 ``/CCLAnnotator/app/Models/Task.php``，同时，为了\
记录任务操作，设计一个动态模型，记录任务动态，路径为 ``/CCLAnnotator/app/Models/Trend.php``。

再来看路由设计：

.. code-block:: php
   
   <?php
   Route::post('task/assign', 'TasksController@assign')->name('task.assign');
   ?>

控制器 ``TasksController`` 也很简单：

.. function:: assign(Request $request)
   :module: TasksController

   向指定用户批量指派任务，并返回视图。

   :param Request: $request 收到客户端发来的HTTP请求

对于任务状态，其存在四个阶段：

============== ================ =======================================================================================
**状态**        **标识**         **备注**
待标注          1                待标注 -> 正在标注
待校对          2                完成标注 -> 待校对 -> 正在校对
待审核          3                完成校对 -> 待审核 -> 正在审核
已存档          4                完成审核 -> 已存档
============== ================ =======================================================================================
